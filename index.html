# ChatGPT처럼 유니티 게임 개발

이 프로젝트는 생성형 AI 기술이 게임 개발 과정에 얼마나 혁신적으로 개입할 수 있는지 실증적으로 보여주는 것을 목표로 합니다. 단순한 스크립트 생성을 넘어, 사용자의 아이디어를 구조화된 데이터와 유니티 컴포넌트로 즉시 변환하여 게임 프로토타입을 제작하는 전 과정을 자동화하고자 합니다. 이 실험을 통해 미래 AI가 구현할 수 있는 영역을 직접 탐색하고 시연합니다.

주요 목표

  • 자연어 프롬프트를 정확한 Unity C# 스크립트로 변환하는 AI 파이프라인 구축.
  • Unity 에디터 내에서 AI 기능을 사용할 수 있는 커스텀 UI/UX 개발.
  • AI 기술의 뛰어난 잠재력과 구현 가능성 입증.

핵심 아키텍처

프로젝트의 핵심은 사용자 프롬프트가 실제 Unity 씬 구성 요소로 변환되는 3단계 워크플로우에 있습니다. 각 단계를 클릭하여 AI의 역할과 데이터 흐름을 상세하게 확인하십시오.

P

Unity Game 실행

프롬프트 입력

사용자가 원하는 게임 아이디어를 자연어(예: "토끼가 당근을 모으는 2D 플랫포머 게임")로 입력하는 단계입니다. 이 입력이 전체 프로세스의 시발점입니다.

A. 프롬프트 분석 및 JSON 구조화 (LLM)

LLM은 프롬프트를 분석하여 게임 타입, 핵심 메커니즘, 필요한 에셋을 포함하는 JSON 스키마로 변환합니다. 이는 후속 단계의 입력으로 사용되는 구조화된 데이터입니다.

JSON 스키마 예시:

{
  "gameType": "2D Platformer",
  "mechanics": [
    {"name": "PlayerMovement", "scriptName": "PlayerController.cs"},
    {"name": "Collectable", "scriptName": "Collectable.cs"}
  ]
}

B. C# 스크립트 생성 (LLM)

이전 단계에서 생성된 JSON의 `mechanics` 목록을 기반으로, LLM이 Unity의 MonoBehaviour를 상속하는 완벽한 C# 스크립트 코드를 생성합니다. 코드에는 주석과 필수 `public` 변수들이 포함됩니다.

스크립트 요구사항:

  • Unity 표준 함수(`Start`, `Update`) 구현
  • 정확한 문법 및 Unity API 사용

C. Unity 통합 프레임워크 (개발자 영역)

개발자가 미리 구축한 프레임워크가 AI가 생성한 C# 파일을 프로젝트 폴더에 저장하고, Unity 씬에 필요한 게임 오브젝트를 자동으로 생성한 뒤, 해당 오브젝트에 스크립트를 AddComponent 합니다.

주요 기능:

  • 스크립트 파일 시스템 통합
  • 씬 오브젝트 자동 생성 및 컴포넌트 부착

Unity Game 실행

모든 스크립트와 씬 구성이 완료되면, 사용자는 Unity 에디터에서 즉시 프로토타입 게임을 실행하여 프롬프트의 결과물을 확인할 수 있습니다.

개발 로드맵

프로젝트는 3단계의 명확한 목표를 가지고 진행됩니다. 각 단계의 제목을 클릭하여 상세 목표와 결과물을 확인하십시오.

핵심 목표

  • 가장 기본적인 단일 스크립트 생성 및 적용 파이프라인 완성.
  • 결과물: 큐브가 좌우로 움직이는 것을 확인할 수 있는 가장 간단한 게임.

핵심 목표

  • 두 개 이상의 스크립트 상호작용 및 에셋 플레이스홀더 처리 구현.
  • 결과물: 점수 획득 기능이 있는 기본 프로토타입 게임.

핵심 목표

  • Unity 에디터 내에서 프롬프트를 입력 및 결과 확인이 가능한 커스텀 UI 개발.
  • 결과물: 'ChatGPT처럼' 아이디어를 바로 입력하고 게임을 생성할 수 있는 완벽한 UX/UI.
